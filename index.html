Ecomon

During the course on innovative mobile services and systems, me and three course mates worked on a project called Ecomon. The assignment was to develop a system using three screens, like DateMe below, that makes the user reflect on personal energy consumption. Our solution was based on gamification and context awareness, and the application sent notification based on the user's context. The user gets points depending on the action taken on the notification, gets achievements and levels up. If it's raining outside, the user might get a notification that says "If you take the bus now, you can make it to work and not take your car". The user can then choose to "Buy a ticket" or "Read more", which gives more or less points in the system. Check out the web app demo. I worked on the Android application.

Tech used: Android native with Wear extension, retrofit, yeoman, MEAN-stack.

Saturn

I was part of the 24-hour Diversify-hackathon by Spotify in the fall of 2014, and with my awesome team mates we created Saturn, a game made to broaden your taste in music. Somewhat based on the swedish game "När då då", you guess in which time span on a time line a random song came out. If you are correct, the song is added to the timeline, and so another time span is added for the next song. The songs are from the top lists from every year since 1950. Try out the Saturn game for yourself! The source is on GitHub. I worked on the front-end design.

Tech used: AngularJS, Spotify API.

DateMe

On a course on innovative mobile services and systems, I got the chance to try out Android for the first time. The assignment was to create a dating service that uses three different screens: a web application and an Android application that includes Android Wear. Users register and change profile details via the web application, and via the Android app, users are notified when they are close to other users. If a user wants to meet, other nearby users can send him or her a message asking to meet up. Via Wear, users can see details on nearby users that wants to meet.The DateMe website is up and running on Heroku.

Tech used: Android native with Wear extension, retrofit, yeoman, MEAN-stack.

Artnet

On a prototyping course for mobile application, me and two classmates worked on a prototype for Artnet, a social network for art. The purpose of the application was to increse interest in art and culture, and to make it more available. With the app, users can easily find art that is closeby or in a museum, and once they visit an art piece, they can rate it. By rating different kinds of art, the user gets achievements and works their way up a title tree. Everyone starts out as a brush cleaner, and goes on to become graffiti guru, culture aunt, etc. The prototype made was a web app. See the Artnet demo or check out the source on GitHub.

Tech used: AngularJS.

SmartOffice

Ubiquitous computing is all about integrating technology into our everyday life, our surroundings and the objects we interact with. During an assignment we had the task of designing a context-aware smart environment, and my group came up with SmartOffice, a system that can basically replace office assistants. We made a scale model of an office using cardboard, and with different sensors and servos we made a functioning prototypes of different functionalities. The project was also documented with two short videos that can be seen on the project website.

Tech used: Phidgets: RFID-tags and readers, LCD-screens, LED-lights, servos etc.

Hashtagz image search

Another course project I did was a mashup of the APIs of Instagram, 500px and Flickr using a MEAN-stack that I built with the help of this awesome scotch.io tutorial. The idea is that you can search for any tag and see the most recent pictures from the different services tagged with that. In the full project you could like images and they were saved and displayed on a separate page.

I did this project mainly to work some more with AngularJS and to try out the MEAN-stack, since I am a true sucker for Single Page Applications. You can try out the search part of the application on the Hashtagz demo page.

Tech used: MEAN-stack - MongoDB (with mongoose.js), Express.js, AngularJS and Node.js.
